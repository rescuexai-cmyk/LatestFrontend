9. Screen-by-Screen Click Actions

Home (/)

  • Find a Ride Now! → /services
  • Open Drivers' App → onboarding / welcome / /driver (by status)
  • Active Ride Banner → /booking/searching or /booking/driver-assigned
  • Switch account → bottom sheet

Services (/services)

  • Profile avatar → /profile
  • Later / Schedule → schedule sheet or /booking/find-trip
  • Service tiles / search / places → /booking/find-trip?autoSearch=true
  • Active ride exists → /booking/driver-assigned or /booking/searching

Find Trip (/booking/find-trip)

  • Book Now → /booking/payment
  • End drawer → Home, History, Payment methods sheet, Help sheet, Profile

Payment (/booking/payment)

  • Slide to confirm → /booking/searching

Searching (/booking/searching)

  • Driver matched → /booking/driver-assigned
  • Cancel → /booking/find-trip
  • Minimize → /services

Driver Assigned (/booking/driver-assigned)

  • Chat → bottom sheet (not a route)
  • Ride complete → rating sheet → /
  • Cancel → /

Profile (/profile)

  • Total Rides → /history
  • Switch to driver → / (driver gateway)
  • Notifications / Help / Saved places → bottom sheets
  • Server config → /settings/server
  • Logout / Delete account → /login

Driver Home (/driver)

  • Accept ride → /driver/active-ride
  • Drawer: Switch to Rider → /services
  • Drawer: History / Earnings / Settings / Help → bottom sheets
  • Drawer: Update Documents → /driver/onboarding?isUpdateMode=true

10. Architecture Notes

  1. / is not the booking home — it is the role gateway. Rider booking hub is /services.
  2. Live rider ride = /booking/driver-assigned (not /ride/:id/tracking, which is legacy).
  3. Login OTP is inline on /login — no route change until verify succeeds.
  4. Chat opens as a bottom sheet on active-ride screens, not /ride/:id/chat.
  5. Active Ride Banner appears on Home, Services, Profile, History.
  6. Driver vs Rider split: Rider hub = /services; Driver hub = /driver.
